# Architecture Decision Records

This directory holds forward-looking architecture decisions. Use it for
decisions that should guide future code changes, especially when the decision
cuts across fetching, parsing, processing, storage, CLI, or UI.

Historical reviews and migration notes stay in `docs/history/`. Current module
truth stays in `docs/structure/` and `docs/feature/`.
